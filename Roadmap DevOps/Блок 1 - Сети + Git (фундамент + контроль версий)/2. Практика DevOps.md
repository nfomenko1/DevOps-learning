1. **git init** - папку devops-learning превратил в в git‑репозиторий.
2. **git remote add origin URL** - связал папку с GitHub.
3. **git add** - добавил файлы в индекс (гит) (Сохранить в истории).
4. **git commit -m "Кратко о изменениях"** - перенес все изменения из индекса в репозиторий (Сохранить локально).
5. **git push origin -u main** - отправил обновленную информацию на GitHub.

> Пояснение для себя: 
- `push` — отправить локальные коммиты на удалённый репозиторий.
    
- `origin` — имя GitHub‑репо, которое мы задали `git remote add origin ...`.
    
- `main` — ветка, которую отправляем.
    
- `-u` — запомнить связку «локальная main ↔ origin/main», чтобы дальше можно было писать просто `git push`
